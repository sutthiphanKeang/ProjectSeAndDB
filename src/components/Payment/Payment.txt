Payment System 
(เป็นทั้งหมดที่แสดงผล อาจจะมีการเพิ่ม attribute บางตัวได้
เช่น ประเภทของใบชำระเงิน พวกค่าเช่า ค่าปรับ ต้องอยู่ที่ต้องสร้างหมายเลขคำสั่งสั่งซื้ออีกที)

(preprocess)
Query Data from DB
***ในส่วนนี้จะรวมการแสดงผลของค่าเช่า และ ค่าปรับด้วย 
*แสดงรายละเอียดที่ต้องจ่าย เน้นที่ราคา
    - รูปรถ
    - หมายเลขใบแจ้งชำระ
    - ราคารถ
    - ราคาประกัน(ที่บังคับจ่าย)
    - ราคาประกันส่วนตัว(ที่เพิ่มเอง)
    - เพิ่มเติม(อาจเป็นส่วนลด หรือ ค่าปรับ)

*แสดงสรุปที่ต้องจ่าย บวกค่่าบริการ
    - ราคารวม(ที่ยังไม่รวม service fee)
    - ราคา service fee
    - ราคารวมทั้งหมด

(main process)
- QR Code (ไม่มีข้อมูล input จากลูกค้า)
    output to gatewat api
        - หมายเลขใบแจ้งชำระ
        - ราคารวมทั้งหมด

- Credit Card
    input 
        - ชื่อบัตร
        - เลขบัตร
        - วันหมดอายุ
        - CVV/CVC

    output to gatewat api
        - หมายเลขใบแจ้งชำระ
        - ราคารวมทั้งหมด
        - ชื่อบัตร
        - เลขบัตร
        - วันหมดอายุ
        - CVV/CVC

Output from Gateway to DB
    - หมายเลขใบแจ้งชำระ
    - สถานะการชำระ (ในที่นี้จะมีแต่ complete เพราะเป็นการ mock แต่จะเขียนดักสถานะ fail ไว้ อาจเป็นเลข 0 กับ 1)
